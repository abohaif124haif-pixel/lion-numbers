شرح مفصّل لنظام "لغة الأرقام الأسدية"

فكرة النظام: يقوم هذا النظام بتحويل كل حرف من حروف اللغة العربية أو الإنجليزية إلى رمز عددي ثابت. الهدف هو تمثيل النصوص بصيغة رقمية قابلة للتخزين أو المعالجة أو التشفير.

جدول الترميز (ملخص): الحروف العربية مُعيّنة أرقامًا من (1) فصاعدًا. الحروف الإنجليزية الصغيرة لها أرقام مُنفصلة، أما الحروف الكبيرة فتميّز ببادئة فاصلة مفردة ' قبل الرقم (مثال: '1 للحرف A الكبير).

الفواصل المستخدمة:

  • -0- بين رموز الحروف داخل الكلمة (يسمح بتمييز الأرقام متعددة الخانات).
  • -00- بين الكلمات.

قواعد التشفير خطوة بخطوة:

  1. قسّم النص إلى كلمات باستخدام المسافات.
  2. لكل كلمة: استخرج الحروف بالترتيب وحول كل حرف إلى رقمه وفق الجدول.
  3. ضع -0- بين كل رمز وآخر داخل نفس الكلمة.
  4. ضع -00- بين الكلمات المختلفة.

أمثلة عملية:

  • النص العربي: الاسد → 1-0-23-0-1-0-12-0-8
  • النص الإنجليزي: Abdul (A كبيرة) → '1-0-2-0-4-0-21-0-12

فك التشفير: لعكس العملية افعل بالعكس: قسم النص على -00- للحصول على الكلمات، ثم قسّم كل كلمة على -0- لتحصل على رموز الحروف، وحوّل كل رمز إلى الحرف المقابل وفق الجدول العكسي.

ملاحظات مهمة:

  • النظام يميّز الحروف الإنجليزية الكبيرة عبر بادئة الفاصلة المفردة. عند تحويل TLON إلى عربي، ستظهر فقط الحروف العربية المشفرة؛ أما الرموز التي تمثل حروفًا إنجليزية فستُتَرجم فقط عند استخدام زر "TLON.to.En".
  • يمكن توسيع الخريطة لتشمل علامات الترقيم أو الأرقام أو الرموز الخاصة إذا رغبت.
  • احرص على استخدام الفواصل الدقيقة -0- و-00- لتفادي الغموض عند فك الشفرة.

وصلى الله على نبينا محمد وعلى آله وصحبه أجمعين.

Detailed Explanation of the "Lion Numbers Language"

Concept: This system maps each Arabic and English letter to a fixed numeric token. The goal is to represent text in a numeric format suitable for storage, processing or lightweight obfuscation.

Encoding summary: Arabic letters are assigned numeric tokens starting from 1. Lowercase English letters have their own numeric tokens, while uppercase English letters are distinguished by a leading apostrophe ' (for example '1 for capital A).

Separators:

  • -0- separates letter tokens inside a word (keeps multi-digit tokens unambiguous).
  • -00- separates words.

Encoding rules (step by step):

  1. Split the text into words by spaces.
  2. For each word: replace each character with its numeric token according to the mapping.
  3. Join tokens inside a word with -0-.
  4. Join words with -00-.

Examples:

  • Arabic word: الاسد → 1-0-23-0-1-0-12-0-8
  • English name: Abdul (capital A) → '1-0-2-0-4-0-21-0-12

Decoding: Split by -00- to get words, then split each word by -0- to get tokens, and map each token back to the corresponding character using the reverse table.

Notes:

  • Uppercase/lowercase English are differentiated via the apostrophe prefix for uppercase tokens.
  • You can extend the mapping to include punctuation, digits, or other symbols.
  • This scheme is useful for lightweight encoding and consistent internal representation across Arabic and English text.
